# Домашня платформа квартири

Цей репозиторій містить дизайн домашнього сервера та єдиної панелі для двох людей. Мета — локальні сервіси, які спрощують побут, а не створюють ще одну роботу з адміністрування.

## Робоча конфігурація

- `compose.yaml` — єдина точка запуску домашнього стеку.
- `dashboard` — власна адаптивна панель «Наш дім» з Dockerfile, API snapshot і тестами.
- `infra/core` — Caddy та локальний TLS.
- `infra/home` — Mealie, Vikunja, Homebox і Vaultwarden з окремими даними.
- `infra/observability` — Uptime Kuma без доступу до Docker socket.
- `.env.example` — версії образів і перелік обов'язкових секретів без реальних значень.
- `scripts` — безпечна генерація секретів та перевірка Compose.

Локальний preview панелі під час розробки: `cd dashboard && npm run dev`. Production-запуск відбувається разом з усім стеком через кореневий Compose та адресу `https://home.home.arpa`.

Для автоматичного налаштування локального DNS і стандартного HTTPS виконайте на сервері `bash scripts/configure-lan-access.sh`, а потім вкажіть IP сервера як LAN/DHCP DNS у роутері. Деталі є в `docs/lan-access.md`.

## Зафіксовані принципи

1. Одна домашня панель для щоденних сценаріїв; адмінки сервісів приховані від звичайного користування.
2. Автоматизація реагує на присутність, час і датчики; ручна кнопка залишається лише для важливих винятків.
3. Особисті дані доступні тільки локально або через VPN.
4. Ігрові сервери можуть споживати багато ресурсів, але не мають впливати на Home Assistant, паролі та резервні копії.
5. RAID/дзеркало забезпечує доступність, але не замінює резервну копію.
