# Домашня панель: продуктовий та UI-дизайн

## Продуктова мета

За 10 секунд відповісти на три питання: що сьогодні важливо, чи все гаразд удома, яка одна наступна дія потрібна. Панель орієнтована на двох мешканців, а не на адміністратора сервера.

## Інформаційна архітектура

### Головна — «Сьогодні»

1. Привітання, дата, погода та статус «удома/поза домом».
2. Один адаптивний блок уваги: протікання, відкрите вікно, термінова задача або продукт із коротким строком.
3. «Наш день»: до трьох задач із Vikunja та план їжі з Mealie.
4. «Швидко зробити»: додати продукт, додати задачу, увімкнути вечірній режим.
5. «Дім»: короткий стан клімату, дверей, води й енергії без переліку кожного датчика.
6. «Відпочинок»: продовжити перегляд та один спільний список бажаного.

### Другорядні розділи

- **План:** задачі, покупки, харчування, нагадування.
- **Дім:** кімнати, клімат, безпека, енергія та сценарії.
- **Речі:** пошук Homebox, гарантії й майбутнє обслуговування.
- **Медіа:** Jellyfin та спільний список перегляду.
- **Сервіси:** технічний стан — доступний лише власнику/адміністратору.

Vaultwarden не вбудовується iframe і не віддає секрети панелі. Кнопка відкриває офіційний клієнт або web vault у новому захищеному контексті.

## Візуальна система

- Характер: теплий, спокійний, побутовий; не «кіберпанк» і не серверна консоль.
- Світла та темна тема за системним налаштуванням; темна тема оптимізована для TV/нічного планшета.
- Основний фон: теплий нейтральний; акцент: приглушений зелений; попередження: бурштиновий; небезпека: червоний лише для дійсно термінових станів.
- Типографіка: системний sans-serif, великі цифри для температури/часу, мінімум дрібного тексту.
- Мінімальна touch-ціль 44×44 px; ключові дії 52–56 px.
- Картки мають відповідати на питання, а не повторювати структуру API.

## Адаптивність

- Телефон: одна колонка, нижня навігація, критична інформація в першому екрані.
- Планшет на стіні: 2–3 колонки, режим без прокручування для основних станів, автоматичне приглушення вночі.
- Desktop: 3 колонки й швидкий пошук.
- TV: read-only огляд і великі фокусовані елементи; введення даних не передбачене.

## Компоненти першої версії

- `AttentionCard`: рівно одна найважливіша подія з поясненням і дією.
- `TodayList`: об’єднані задачі й побутові плани, максимум п’ять рядків.
- `MealCard`: сьогоднішня страва, відсутні продукти, перехід у Mealie.
- `HomeSummary`: агрегований стан «Усе гаразд» або перелік винятків.
- `QuickCapture`: одна форма, яка маршрутизує запис у список продуктів або задачі.
- `SceneButton`: підтвердження лише для небезпечних дій; оптимістичний стан для світла/сцен.
- `ServiceHealth`: прихований адмінський компонент без технічних деталей на головній.

## Стани та помилки

- Якщо інтеграція недоступна, показувати останній відомий стан із часом, а не нульові/неправдиві дані.
- Відмова одного сервісу не блокує всю панель.
- Небезпечні дії — замки, вода, сигналізація — мають явний результат і timeout.
- Панель ніколи не показує токени, паролі, внутрішні IP або stack traces.
- Усі основні дії доступні клавіатурою та мають текстові labels, не лише іконки.

## Контракти інтеграцій

Власний backend-for-frontend надає панелі нормалізовані read models: `Today`, `HomeSummary`, `Attention`, `Meal`, `InventoryAlert`. Токени зовнішніх сервісів зберігаються тільки server-side. Запис дозволений через окремі вузькі команди: створити задачу, додати продукт, активувати дозволену сцену; загальний proxy до API заборонений.
